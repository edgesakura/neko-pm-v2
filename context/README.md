# context ディレクトリ

プロジェクト固有のコンテキストを管理するディレクトリにゃ。

## 目的
- プロジェクトごとの知識・決定事項を保存
- セッション間での情報共有
- 新規参加者（子猫）への引継ぎ

## ファイル構成
```
context/
  README.md           ← このファイル
  template.md         ← プロジェクトコンテキストのテンプレート
  {project_id}.md     ← プロジェクト固有のコンテキスト
```

## 使い方

### 新規プロジェクト追加時
1. `template.md` をコピーして `context/{project_id}.md` を作成
2. テンプレートに沿って記載

### 子猫の作業開始時
タスクYAMLに `project: {project_id}` が指定されている場合:
1. `CLAUDE.md` を読む（プロジェクト全体のルール）
2. `memory/global_context.md` を読む（ご主人の好み・システム方針）
3. `context/{project_id}.md` を読む（プロジェクト固有情報）

## テンプレートの使い方

`template.md` には以下の項目が含まれているにゃ：
- **What（これは何か）**: プロジェクトの概要
- **Why（なぜやるのか）**: 目的と成功の定義
- **Who（誰が関係するか）**: 責任者・関係者
- **Constraints（制約は何か）**: 期限・予算など
- **Current State（今どこにいるか）**: ステータス・進捗
- **Decisions（決まったこと）**: 重要な意思決定の記録
- **Notes（メモ・気づき）**: 自由記述

## 更新ルール
- 重要な決定があったら即座に更新にゃ
- 最終更新日を必ず更新にゃ
- 不要になった情報は削除（シンプルに保つ）にゃ
- ボスねこまたは番猫が更新責任を持つにゃ〜

## 注意事項
- 機密情報（API キー、パスワード等）は記載しないにゃ
- Git 管理外の情報が必要な場合は `memory/global_context.md` に記録にゃ
- プロジェクト横断の知見は `memory/global_context.md` に記録にゃ〜
